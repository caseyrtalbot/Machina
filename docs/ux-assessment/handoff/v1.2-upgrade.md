# v1.2 — Upgrade Arc (Handoff)

**Goal:** move the reviewer verdict from "impressive" to "ship." v1.1 sanded the seams; v1.2 rebuilds the weak surfaces and wires the signature systems (tensions, ghosts, graph) to actually route signal to action.

**Scope:** 6 themed batches — shared state primitives, Health redesign, Sidebar contextualization, Tension anchoring, Graph completeness, Ghosts-to-action.

**Budget:** ~200 eng-hours / 4–6 weeks.

**Branch model:** feature branch per week (`ux/v1.2-w1-state-primitives`, `ux/v1.2-w2-health`, …). Each week ships its own PR to `main`. Parallel work OK if two engineers; single-thread if solo.

## 1. Decisions that must be made before code lands

Each batch starts with a decision gate. The engineer does not choose these — the user does. Surface the options, wait for the call, then execute.

### Decision D1 — Sidebar model (before Week 3)

**Options:**
- **D1a. Collapsible per-section.** Daily Notes / Tags / Files / Tensions become independently collapsible. User configures once, persists. Low risk.
- **D1b. Contextual to active panel.** Sidebar rotates content based on active panel. Canvas shows filters + recent; Graph shows legend + clusters; Ghosts shows tier filters; Editor shows file tree + outline. High reward, higher risk.
- **D1c. Hybrid.** File tree always persists; a second pane below is contextual. Safest middle ground.

**Recommendation:** start **D1a** for v1.2, ship **D1b/c** as v1.3 experiment behind a feature flag. Get user confirmation.

### Decision D2 — Tension anchoring visual (before Week 4)

**Options:**
- **D2a. Amber glow on participating cards** + amber edge between them on canvas. Uses existing `ARTIFACT_COLORS.tension`, `EDGE_KIND_COLORS.tension` tokens.
- **D2b. Dedicated tension-mode overlay** — clicking a tension triggers a filtered canvas view showing only participating cards + context.
- **D2c. Both.** Default = always-on glow at 30% opacity; click tension → full mode overlay.

**Recommendation:** **D2c**. Defer the mode overlay if Week 4 runs tight — glow alone is high-value.

### Decision D3 — Ghost promotion UX (before Week 6)

**Options:**
- **D3a. One-click create with default template** — promotes to a note in `ghosts/` folder with a stub.
- **D3b. Template picker** — user picks which template (note / concept / pattern / …) on promotion.
- **D3c. Canvas-first** — promoted ghost drops onto the current canvas as a new card; user fills it in-place.

**Recommendation:** **D3a** with a follow-up Settings option to change the default folder. Keep the template picker for v1.3.

## 2. Weekly batches

### Week 1 — Shared state primitives (theme T3)

**Findings addressed:** F-static-10, F-static-11, F-cross-06

#### Deliverables

1. **`<PanelState>` primitive** — `src/renderer/src/design/components/PanelState.tsx`
   - Props: `kind: 'empty' | 'loading' | 'error'`, `title`, `description`, `icon?`, `action?: { label; onClick }`
   - Renders centered, ≤480px wide, tasteful typography matching Ghosts hero pattern
   - Loading variant: subtle animated indicator (respects `useReducedMotion`)
   - Error variant: uses `colors.claude.error` accent
2. **`PanelErrorBoundary` redesign** — `src/renderer/src/components/PanelErrorBoundary.tsx`
   - Replace default fallback with `<PanelState kind="error">` composition
   - Copy: "Machina hit a snag in <panel name>. {{errorMessage}}. We logged the details. [Retry] [Report]"
   - `[Report]` opens a prefilled issue template (copy stack + repro hint to clipboard, open GitHub issues URL)
3. **Panel-purpose copy rollout** (F-cross-06)
   - One-line purpose string at the top of every empty panel state. Dismissible.
   - `PanelPurpose` component or inline copy within each panel's empty state.
   - Copy examples:
     - Canvas: "Arrange notes, agents, and tools in space. Drag notes from the sidebar, or right-click to add."
     - Graph: "How your vault connects itself. Hover to isolate, drag to compare clusters."
     - Ghosts: "Ideas you've referenced but haven't written yet. The higher the count, the more it matters."
     - Health: "Issues Machina found while reading your vault. Fix, ignore, or teach us to skip."
     - Workbench: "Live sessions from your agents. Monitor output, review milestones, resume or stop."

#### Definition of done

- `<PanelState>` exported from `design/components/index.ts` (create the barrel if missing)
- Canvas, Graph, Ghosts, Workbench, Health all use it for their empty/loading/error states
- Purpose copy visible on all five panels when empty; persistent dismiss via a per-panel `ui-store` key
- PR passes `npm run check`
- Pass 2 re-capture of every panel empty state — update rubric scores

### Week 2 — Health panel redesign (theme T4)

**Findings addressed:** F-health-01, -02, -03, -04, -06 (F-health-05 and -07 were v1.1)

This is the single biggest product surface fix in v1.2. Budget the full week; do not dilute with other work.

#### Deliverables

1. **Copy rewrite** — every issue becomes `{{friendly name}} / {{why it matters}} / {{action}}`. Examples:
   - Parse error → "Can't read Architecture.md. This note won't appear in your graph or search. [Open file] [Retry] [Ignore]"
   - File not in worker index → "Casey Central.base isn't indexed. It won't match search results. [Reindex] [Ignore file type]"
   - Broken reference → "claude-code-source-analysis1.5 is referenced from 3 notes but doesn't exist. [Create note] [Remove references] [Show notes]"
2. **Severity encoding** (F-health-02): three-tier visual — Critical (red) / Warning (amber) / Info (muted). Dot or left-edge bar; severity count in header.
3. **Hero-number header** (F-health-03): echo GhostPanel pattern — "2 critical · 319 warnings · 0 info" with the biggest digit on the left.
4. **Group by file** (F-health-04): collapse related issues under the file they're in. Expand on click. Filter panel-level actions at the file level.
5. **Filter bar** (F-health-06): severity filter + folder filter + free-text search. Compact top-of-panel strip.

#### Definition of done

- Health panel no longer reads like a log file
- Every issue has at least one `[primary action]` button
- Severity dots/bars use tokens from v1.1 (`status.critical/warning/info`)
- Zero-issues state exists and celebrates — "Vault is clean."
- A non-developer user can read any issue row and understand what's wrong and what to do
- PR passes `npm run check`
- Pass 2 rescoring of Health panel jumps from current 1/5 and 2/5s to ≥4/5 on IA, Interaction design, UX copy, Color & theming

### Week 3 — Sidebar contextualization (theme T5)

**Findings addressed:** F-cross-01 (primary), F-cross-07 reinforcement

**Prereq:** Decision D1 made.

#### Deliverables (assuming D1a — collapsible)

1. Each sidebar section (Tensions, Daily Notes, Tags, Files) becomes independently collapsible with a `<SectionLabel>` + caret header.
2. Collapse state persists to `ui-store` (per-user) and rehydrates on launch.

4. Tensions section gets its own empty/populated state treatment using `<PanelState>` from Week 1.

More scope. Sidebar content routes through a `<SidebarSlot panel={active}>` primitive; each panel declares its sidebar content. Scope carefully — budget +3 days.

#### Definition of done

- Collapse/expand states persist
- Tensions section anchored to sidebar (visual prep for Week 4)
- PR passes `npm run check`

### Week 4 — Tension anchoring (theme T9)

**Findings addressed:** F-canvas-11, F-cross-09, F-canvas-08 (edges kind-colored as prereq)

**Prereq:** Decision D2 made.

#### Deliverables

1. **Connection edge kind colors** (F-canvas-08): apply `EDGE_KIND_COLORS` token map to canvas edges. Connection-type edges in slate, causal in pink, imports in blue, etc. Arrowheads for directional edges (`derived_from`, `references`, `imports`, `causal`).
2. **Edge hover emphasis:** hovering any card fades unrelated edges to 10%; edges connected to the card go to 100%. Hovering an edge itself shows a tooltip with kind + endpoints.
3. **Tension glow on participating cards:** each card participating in an active tension gets a subtle amber outer ring (`ARTIFACT_COLORS.tension` at 40% opacity, 2px). Rest at 30% by default.
4. **Tension-click focus:** clicking a row in the sidebar Tensions section:
   - Pans/zooms canvas to frame the participating cards
   - Renders an amber edge (`EDGE_KIND_COLORS.tension`) between them if not already present
   - Shows an inline tension description as a floating card between them (using `floatingPanel.glass` material)
   - Provides [Dismiss] / [Resolve] / [View full tension note] actions
5. **Canvas mutation safety:** any tension-related visual state is rendered at the React/Pixi overlay layer — it does not touch the persisted canvas file. No `canvas:apply-plan` calls for tension visualization.

#### Definition of done

- Edges are visually meaningful (kind-colored, arrowed where directional)
- Hovering cards shows connection context
- Tensions visible on canvas, anchored to specific cards
- Clicking a sidebar tension row focuses the canvas
- PR passes `npm run check`
- Pass 2 rescoring: Canvas visual-hierarchy + consistency scores jump; signature feature (tensions) becomes demonstrable in 10 seconds

## 5. Risk register

| Risk | Likelihood | Mitigation |
|------|------------|------------|
| Health redesign copy takes longer than expected | High | Use Notion / Linear empty-state libraries as reference; draft all copy before writing any code |
| Sidebar contextualization breaks existing user habits | Medium | Ship D1a first (collapsible-only) behind a small migration note; gather feedback before D1b |
| Tension-click focus mutates canvas unintentionally | High | Strict overlay-only rendering; no `canvas:apply-plan` calls. Validate with an integration test |
| Ghost promotion writes files to an unexpected vault location | Medium | Default folder configurable via settings; path shown in confirmation flash |
| `<PanelState>` primitive fails to cover every existing empty state | Low | Keep it composable; accept a `children` override for edge cases |
| Graph legend flickers or stutters with 467+ nodes | Low | Render legend separately from graph; it should be static per data, re-render only on filter |
