#### Task 1.2 — Global `:focus-visible` treatment [F-static-07, S effort]

- **Why:** Pass 1 found only 4 files using `focus-visible`/`focus:ring`/`outline:`. Browser default focus rings are invisible on dark glass surfaces.
- **Approach:**
  1. Add to `src/renderer/src/assets/index.css`:

     ```css
     /* Global keyboard focus affordance — applies only on keyboard navigation */
     :where(
       button,
       a,
       input,
       select,
       textarea,
       [role='button'],
       [role='menuitem'],
       [role='option'],
       [tabindex]
     ):focus-visible {
       outline: 2px solid var(--color-accent-default);
       outline-offset: 2px;
       border-radius: var(--radius-inline, 4px);
     }

     @media (prefers-reduced-transparency: reduce) {
       :where(
         button,
         a,
         input,
         select,
         textarea,
         [role='button'],
         [role='menuitem'],
         [role='option'],
         [tabindex]
       ):focus-visible {
         outline-width: 3px;
       }
     }
     ```

  2. Audit any `outline: none` / `outline: 0` in css **and** in JSX inline styles (`:where()` has specificity 0, so any `outline: none` wins even without `!important`). Either remove (preferred) or pair with a custom `:focus-visible` rule.
     ```bash
     rg -n 'outline:\s*(none|0)' src/renderer/src                                    # css + tsx text
     rg -n '[oO]utline.*none' --glob '*.tsx' src/renderer/src                        # JSX inline styles
     ```

- **Definition of done:**

#### Task 1.4 — `prefers-reduced-motion` strategy [F-static-09, M]

- **Why:** P1 accessibility gap per roadmap §2. macOS Reduce-motion users currently get full graph physics and canvas tween animation.
- **Approach:**
  1. Create `src/renderer/src/hooks/useReducedMotion.ts`:
     ```ts
     import { useEffect, useState } from 'react'
     export function useReducedMotion(): boolean {
       const [reduced, setReduced] = useState(
         () =>
           typeof window !== 'undefined' &&
           window.matchMedia('(prefers-reduced-motion: reduce)').matches
       )
       useEffect(() => {
         const mq = window.matchMedia('(prefers-reduced-motion: reduce)')
         const handler = (e: MediaQueryListEvent) => setReduced(e.matches)
         mq.addEventListener('change', handler)
         return () => mq.removeEventListener('change', handler)
       }, [])
       return reduced
     }
     ```
  2. In `assets/index.css`, add a **scoped** CSS fallback. Do NOT use the `*, *::before, *::after` wildcard — it kills Tiptap cursor blink and CodeMirror internal transitions, degrading the editor experience for reduced-motion users. Instead, target the animation class prefix the design system uses:
     ```css
     @media (prefers-reduced-motion: reduce) {
       [class*='te-'][class*='animate'],
       .te-fade,
       .te-slide,
       .te-pulse {
         animation-duration: 0.01ms !important;
         animation-iteration-count: 1 !important;
         transition-duration: 0.01ms !important;
       }
       html {
         scroll-behavior: auto !important;
       }
     }
     ```
     Audit `assets/index.css` and `design/tokens.ts` for the actual `te-*` animation class names before finalizing this rule.
  3. In Pixi graph physics and canvas tween sites, use `useReducedMotion()` to skip or flatten animations. Grep for `requestAnimationFrame`, `d3-force.tick`, `gsap`, or any animation calls in `src/renderer/src/workers/` and `panels/canvas/`.
  4. Add `src/renderer/src/hooks/__tests__/useReducedMotion.test.ts` — test that the hook reads initial state from `matchMedia`, and responds to `change` events.
- **Definition of done:**
  - `useReducedMotion` hook exists with a passing unit test.
  - Scoped CSS override is in place (no wildcard selectors).
  - At least the graph force settling and canvas tween sites check the hook.
  - macOS Settings → Accessibility → Reduce motion enabled → graph and canvas animations flatten; Tiptap cursor blink remains intact.
